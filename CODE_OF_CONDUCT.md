# Code of Conduct

## Expected behavior

- Be direct, factual, and respectful.
- Discuss technical tradeoffs with evidence.
- Assume good faith while still naming concrete risks.
- Keep security and user safety concerns visible.

## Unacceptable behavior

- Harassment, threats, or personal attacks.
- Publishing private data, credentials, screenshots, or logs without consent.
- Encouraging misuse of local desktop automation, including CAPTCHA bypass or unauthorized account actions.
- Repeatedly ignoring maintainer safety boundaries.

## Enforcement

Maintainers may remove content, close discussions, block contributors, or reject
changes that violate this code of conduct. Serious safety or privacy issues
should be handled through the security reporting path in `SECURITY.md`.
